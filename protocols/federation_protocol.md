# Federation Protocol

## Purpose

To allow semi-independent seed nodes, libraries, farms, and councils to share structured information without surrendering autonomy.

## Federation principles

- local autonomy first
- shared standards where useful
- no forced centralization
- portable records
- transparent versioning
- explicit provenance

## A federated node may be

- a seed library
- a local guild
- a farm
- a bioregional repository
- an agentic hub
- a council archive

## Federation minimums

A node should publish:
- supported schema versions
- contact or coordination method
- data sharing scope
- provenance policy
- governance context

## Exchange model

Nodes may exchange:
- accession summaries
- stewardship events
- council decisions
- adaptation notes

## Non-goal

Federation is not centralized ownership.

It is interoperability with autonomy.
