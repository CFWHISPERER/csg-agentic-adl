# Architecture

## Overview

This repository supports a polycentric stewardship architecture with five main layers:

1. Seeds and Ecology
2. Local Seed Stewards
3. Regional Seed Guilds
4. Fractal Sortition Councils
5. Agentic AD Hub / Knowledge Commons

## Conceptual flow

Seeds and ecological observations generate stewardship events.

Stewardship events are created by local stewards, seed libraries, farms, and monitoring systems.

Regional guilds aggregate and interpret these events within a bioregional context.

Fractal sortition councils review proposals, tensions, and coordination questions at multiple scales.

Agentic systems help move information between layers without replacing human stewardship or governance.

## Design assumptions

- ecological realities are place-based
- governance should be distributed
- event records should be portable
- agents should be constrained and auditable
- data standards should be simple enough to use in low-resource settings

## High-level entities

- Seed Accession
- Stewardship Event
- Bioregion
- Council Decision
- Exchange Proposal
- Agent Definition

## Trust model

The system assumes:
- not all actors are equally trusted
- claims should be attributable
- decisions should be reviewable
- agents should not silently mutate records
- provenance matters

## Relationship to Perma-Ledger / farmOS

This repository is not intended to replace farmOS or Perma-Ledger.

It is intended to define interoperable structures that could exchange data with those systems through:
- exports
- APIs
- event logs
- adapters

## Long-term aim

To create an open protocol layer for bioregional seed stewardship and adaptive governance.
