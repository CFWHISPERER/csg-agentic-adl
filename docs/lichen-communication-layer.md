# The Lichen Communication Layer

## Overview

The Lichen Project introduces a model of communication that mirrors living systems.

It is not based on:
- centralized infrastructure
- fixed networks
- uniform devices

Instead, it is based on:

- proximity
- routing
- adaptability
- coexistence across differences

> Communication becomes a property of the system, not a service it depends on.

---

## The Problem

In most modern systems:

- two nearby devices communicate through distant infrastructure
- messages leave the local environment and return
- communication depends on external providers

This introduces:

- latency
- fragility
- loss of sovereignty
- unnecessary complexity

---

## The Lichen Model

In the Lichen model:

- nodes communicate directly when possible
- networks form dynamically
- communication adapts to available pathways

This is similar to:

- mycelial networks
- ecological signaling
- watershed flow patterns

---

## Key Properties

### Local First

Communication begins locally.

- nearby nodes connect directly
- no assumption of internet access
- local knowledge remains local unless shared

---

### Hardware Agnostic

Different devices can communicate without sharing the same hardware.

Examples:
- LoRa ↔ Wi-Fi
- Wi-Fi ↔ Ethernet
- radio ↔ wired links

This is possible because:

> the network exists above the transport layer

---

### Bridge Nodes

Some nodes act as bridges.

- they connect multiple interfaces
- they allow dissimilar systems to communicate
- they do not control the network

A single bridge node can enable:

- a field radio to reach a local server
- a phone to reach a remote seed vault
- a disconnected network to rejoin others

---

### Opportunistic Connectivity

The system does not require constant connectivity.

Instead, it relies on:

- moments of connection
- periodic synchronization
- local persistence

This allows operation in:

- rural environments
- disaster scenarios
- low-infrastructure contexts

---

### Sovereign Identity

Each node is self-identifying.

- no central registry
- no dependency on assigned addresses
- identity persists across connections

---

### Embedded Security

Communication is:

- encrypted
- authenticated
- private by design

Security is not added later—it is part of the foundation.

---

## Example

A simple interaction:

1. A steward scans a seed QR code
2. The phone retrieves local data
3. The phone syncs with a nearby Raspberry Pi
4. The Pi later syncs with a seed vault node
5. Data propagates without centralized infrastructure

At no point is a global service required.
